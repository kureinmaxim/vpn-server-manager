# Инструкция по запуску приложения в терминале

Этот файл содержит основные команды для работы с проектом через терминал.

## Основные команды

### 1. Переход в директорию проекта

Прежде всего, вам нужно перейти в папку с проектом. Скопируйте и вставьте эту команду в терминал:

```bash
cd /Users/olgazaharova/Project/ProjectPython/VPNserverManage
```

### 2. Активация виртуального окружения

Для корректной работы приложения необходимо активировать виртуальное окружение.

```bash
source venv/bin/activate
```
После активации вы увидите `(venv)` в начале строки терминала.

### 3. Запуск приложения

После активации окружения, используйте эту команду для запуска приложения:

```bash
python3 app.py
```

## Единая команда для запуска

Вы можете объединить все шаги в одну команду для быстрого запуска:

```bash
cd /Users/olgazaharova/Project/ProjectPython/VPNserverManage && source venv/bin/activate && python3 app.py

Для остановки приложения, запущенного в терминале, просто нажмите комбинацию клавиш:

## Продвинутые команды

### Генерация ключа шифрования

В проекте есть скрипт `generate_key.py` для создания секретного ключа, который используется для шифрования паролей и других чувствительных данных. Этот ключ сохраняется в файле `.env`.

**Важно:** Запускайте этот скрипт только при первой настройке проекта или если вы целенаправленно хотите сменить ключ. Смена ключа сделает все ранее зашифрованные данные нечитаемыми.

Чтобы сгенерировать ключ, выполните команду:

```bash
# Убедитесь, что ваше виртуальное окружение активировано
python3 generate_key.py
```
*Скрипт создаст (или перезапишет) файл `.env` с новым ключом.*

### Принудительная остановка приложения (kill)

Иногда приложение может "зависнуть" и не реагировать на `Ctrl + C`. В таких случаях можно использовать команду `kill`, чтобы принудительно завершить процесс.

**Шаг 1: Найдите ID процесса (PID)**

Сначала нужно узнать PID вашего запущенного приложения. Используйте одну из следующих команд:

```bash
# Способ 1: Найти все процессы python и отфильтровать по имени файла
ps aux | grep 'app_qt.py'

# Способ 2: Использовать pgrep для поиска по имени процесса
pgrep -f app_qt.py
```
Вы увидите строку с информацией о процессе, и одно из чисел (обычно первое) будет его PID.

**Шаг 2: Завершите процесс**

Используйте найденный PID с командой `kill`:

```bash
# Замените <PID> на реальный идентификатор процесса
kill <PID>
```

Если процесс все еще не завершается, можно использовать более "жесткий" сигнал `SIGKILL`:

```bash
# Это почти всегда работает, но используйте с осторожностью
kill -9 <PID>
```

### Отладка приложения

Вы можете запустить приложение в режиме отладки с помощью встроенного отладчика Python (pdb). Это позволит вам пошагово выполнять код.

```bash
# Убедитесь, что ваше виртуальное окружение активировано
python3 -m pdb app_qt.py
```
После запуска вы окажетесь в консоли отладчика, где сможете использовать команды, такие как `n` (next line), `c` (continue), `p <variable>` (print variable).

## Команды для диагностики сервера (Linux)

Эти команды полезны для быстрой диагностики Linux-сервера, на котором может быть развернут VPN. Большинство из них читают системные файлы из директорий `/proc` и `/etc`.

### Основная информация о системе
```bash
# Версия дистрибутива
cat /etc/os-release

# Версия ядра и системы
cat /proc/version

# Краткая информация о дистрибутиве
cat /etc/issue
```

### Информация о процессоре
```bash
# Подробная информация о CPU (может быть длинный вывод)
cat /proc/cpuinfo | cat

# Только модель процессора
cat /proc/cpuinfo | grep "model name" | head -1
```

### Информация о памяти
```bash
# Подробная информация о памяти
cat /proc/meminfo | cat

# Только общий объем памяти
cat /proc/meminfo | grep MemTotal
```

### Информация о дисках
```bash
# Разделы дисков
cat /proc/partitions

# Смонтированные файловые системы
cat /proc/mounts | cat
```

### Другая полезная информация
```bash
# Время работы системы
cat /proc/uptime

# Средняя загрузка системы
cat /proc/loadavg

# Имя хоста
cat /etc/hostname
```

### Комбинированный вывод для быстрой сводки
```bash
echo "=== Система ===" && cat /etc/os-release | head -2 && \
echo "=== Процессор ===" && cat /proc/cpuinfo | grep "model name" | head -1 && \
echo "=== Память ===" && cat /proc/meminfo | grep MemTotal
```

**💡 Совет:** Файлы в `/proc` и `/etc` содержат основную системную информацию. Для более удобного и читаемого вывода можно использовать стандартные утилиты, описанные ниже.

---

## Расшифровка стандартных утилит диагностики

### `uname -a` - Вся информация о ядре

Команда `uname` (unix name) выводит информацию о системе. Флаг `-a` (all) показывает все доступные данные:

```bash
uname -a
```
**Пример вывода:**
`Linux my-server 5.15.0-78-generic #85-Ubuntu SMP ... x86_64 x86_64 x86_64 GNU/Linux`

**Что это значит:**
- `Linux`: Имя ядра.
- `my-server`: Имя хоста (сетевое имя машины).
- `5.15.0-78-generic`: Версия релиза ядра.
- `#85-Ubuntu SMP ...`: Дополнительная информация о сборке, дистрибутиве и поддержке многопроцессорности (SMP).
- `x86_64`: Архитектура процессора (64-битная).

### `lscpu` - Информация о процессоре

Утилита `lscpu` (list cpu) выводит детальную информацию об архитектуре CPU в удобном формате.

```bash
lscpu | cat
```
**Ключевые поля в выводе:**
- `Architecture`: Архитектура (e.g., x86_64).
- `CPU op-mode(s)`: Режимы работы (32-bit, 64-bit).
- `CPU(s)`: Общее количество ядер.
- `Model name`: Полное название модели процессора.
- `Vendor ID`: Производитель (e.g., GenuineIntel, AuthenticAMD).
- `L1, L2, L3 cache`: Размеры кэш-памяти разных уровней.

### `free -h` - Информация об оперативной памяти

Команда `free` показывает количество свободной и используемой оперативной (RAM) и swap-памяти. Флаг `-h` (human-readable) делает вывод читаемым (в килобайтах, мегабайтах, гигабайтах).

```bash
free -h
```
**Пример вывода:**
```
              total        used        free      shared  buff/cache   available
Mem:          7.7Gi       2.1Gi       4.2Gi       1.0Mi       1.4Gi       5.3Gi
Swap:         2.0Gi          0B       2.0Gi
```
- `total`: Общий объем памяти.
- `used`: Сколько памяти используется.
- `free`: Сколько памяти полностью свободно.
- `buff/cache`: Память, используемая ядром для буферов и кэширования.
- `available`: Память, доступная для запуска новых приложений (самый важный показатель).

### `df -h` - Информация о дисковом пространстве

Команда `df` (disk free) показывает использование дискового пространства для всех смонтированных файловых систем. Флаг `-h` (human-readable) также преобразует размеры в удобный формат.

```bash
df -h
```
**Пример вывода:**
```
Filesystem      Size  Used Avail Use% Mounted on
/dev/sda1        50G   20G   28G  42% /
tmpfs           3.9G     0  3.9G   0% /dev/shm
```
- `Filesystem`: Имя раздела диска.
- `Size`: Общий размер раздела.
- `Used`: Использованное пространство.
- `Avail`: Доступное пространство.
- `Use%`: Процент использования.
- `Mounted on`: Точка монтирования (куда "подключен" раздел в файловой системе).
```

## Краткая шпаргалка по диагностике сервера (Debian/Ubuntu)

Этот раздел объединяет наиболее полезные команды для быстрой оценки состояния сервера.

```bash
echo "=== Система ===" && cat /etc/os-release | head -2 && \
echo "=== Процессор ===" && cat /proc/cpuinfo | grep "model name" | head -1 && \
echo "=== Память ===" && cat /proc/meminfo | grep MemTotal
```

### Список установленных пакетов
```bash
# Выведет список всех установленных пакетов (может быть очень длинным)
dpkg -l | grep ii | cat
```

### Ключевые утилиты
```bash
# Информация о ядре и системе
uname -a

# Информация о процессоре
lscpu | cat

# Использование оперативной памяти
free -h

# Использование дискового пространства
df -h
```
