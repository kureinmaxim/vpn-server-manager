# Структура Проекта VPN Server Manager

Этот документ описывает организацию файлов и папок в проекте, а также основной алгоритм работы приложения.

## Дерево Проекта

```
VPNserverManage/
│
├── app.py                      # Основной файл приложения (сервер Flask и логика GUI)
├── build_macos.py              # Скрипт для сборки .app и .dmg для macOS
├── build_windows.py            # Скрипт для сборки .exe для Windows (аналогично)
├── requirements.txt            # Список зависимостей Python
├── config.json                 # Конфигурация приложения (версия, URL-ы)
├── VPNServerManager.spec       # Файл конфигурации для PyInstaller
├── PROJECT_STRUCTURE.md        # Этот файл
│
├── data/
│   ├── servers.json.enc        # Зашифрованный файл с данными о серверах
│   └── hints.json              # Подсказки для команд (шпаргалка)
│
├── static/
│   ├── css/style.css           # Пользовательские стили для интерфейса
│   ├── images/icon.png         # Иконка приложения в формате PNG
│   └── favicon.ico             # Иконка для веб-интерфейса
│
├── templates/
│   ├── layout.html             # Базовый HTML-шаблон (навигация, футер)
│   ├── index.html              # Главная страница (список серверов)
│   ├── add_server.html         # Форма добавления нового сервера
│   ├── edit_server.html        # Форма редактирования сервера
│   ├── about.html              # Страница "О программе"
│   ├── help.html               # Страница справки (куда будет добавлена эта информация)
│   └── ...                     # Другие HTML-страницы
│
└── venv/                       # Виртуальное окружение Python (изолированные зависимости)
```

## Описание Файлов и Директорий

-   **`app.py`**: Сердце приложения. Содержит:
    -   Веб-сервер на базе **Flask**, который обрабатывает все запросы (показать, добавить, удалить сервер).
    -   Логику шифрования/дешифрования данных с помощью ключа из файла `.env`.
    -   Интеграцию с **PyWebView** для отображения веб-интерфейса в нативном окне macOS/Windows.
    -   Функцию корректного завершения работы (`/shutdown`) для освобождения порта.

-   **`build_macos.py` / `build_windows.py`**: Сборочные скрипты. Они автоматизируют процесс упаковки Python-кода в готовое к распространению приложение (`.app` или `.exe`) с помощью **PyInstaller**. Они также обновляют дату в `config.json` перед сборкой.

-   **`requirements.txt`**: Стандартный файл, перечисляющий все библиотеки Python, необходимые для работы проекта (`Flask`, `pywebview` и т.д.).

-   **`config.json`**: Файл настроек. Хранит информацию о версии, разработчике, а также URL-адреса внешних сервисов (например, для проверки IP).

-   **`data/`**: Директория для хранения данных.
    -   `servers.json.enc`: Самый важный файл данных. Это **зашифрованный** JSON-файл, содержащий всю информацию о серверах. Шифрование обеспечивает безопасность.
    -   `hints.json`: Простой JSON-файл для хранения команд-шпаргалок.

-   **`static/`**: Статические ресурсы, которые используются в веб-интерфейсе — CSS-стили, картинки, иконки.

-   **`templates/`**: HTML-шаблоны, которые Flask использует для генерации веб-страниц.

## Алгоритм Работы Программы

1.  **Запуск**: Пользователь запускает приложение (`VPNServerManager.app`).
2.  **Инициализация**:
    -   `app.py` запускает веб-сервер **Flask** в фоновом потоке. Сервер начинает "слушать" локальный адрес (`http://127.0.0.1:5050`).
    -   Сразу после этого **PyWebView** создает нативное окно приложения.
    -   В этом окне загружается веб-страница по адресу сервера Flask.
3.  **Отображение данных**:
    -   Браузер в окне запрашивает главную страницу (`/`).
    -   Flask получает запрос, загружает зашифрованный файл `data/servers.json.enc`, расшифровывает его с помощью ключа из `.env` и передает список серверов в HTML-шаблон `index.html`.
    -   Пользователь видит готовый список своих серверов.
4.  **Взаимодействие**: Любое действие пользователя (нажатие кнопки "Редактировать", "Добавить") отправляет запрос на сервер Flask, который обрабатывает его, изменяет данные, снова шифрует их и сохраняет обратно в `servers.json.enc`.
5.  **Завершение работы**:
    -   Пользователь нажимает на красный крестик закрытия окна.
    -   PyWebView перехватывает это событие и отправляет фоновый запрос на специальный адрес `/shutdown`.
    -   Flask, получив этот запрос, инициирует остановку всего приложения, корректно завершая свою работу и освобождая сетевой порт.
