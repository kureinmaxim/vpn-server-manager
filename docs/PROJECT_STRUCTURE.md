## Описание Файлов и Директорий

### Основные файлы

-   **`app.py`**: Сердце приложения. Содержит:
    -   Веб-сервер на базе **Flask**, который обрабатывает все запросы (показать, добавить, удалить сервер).
    -   Логику шифрования/дешифрования данных с помощью ключа из файла `.env`.
    -   Интеграцию с **PyWebView** для отображения веб-интерфейса в нативном окне macOS.
    -   Функцию корректного завершения работы (`/shutdown`) для освобождения порта.
    -   **⭐ НОВОЕ**: Расширенные функции импорта/экспорта:
        -   `/data/export` - экспорт данных (.enc файл)
        -   `/data/export_key` - экспорт ключа шифрования (.env файл) 
        -   `/data/export_package` - полный экспорт (ZIP с данными, ключом и файлами)
        -   `/data/import_external` - импорт с объединением из другой установки
    -   **🆕 ОФЛАЙН РЕЖИМ**: Улучшенная обработка ошибок и проверка интернета:
        -   Проверка доступности интернета через `socket.create_connection()`
        -   Индикатор состояния сети в интерфейсе
        -   Graceful обработка ошибок при отсутствии подключения
        -   Отключение функций, требующих интернет

-   **`build_macos.py`**: Сборочный скрипт для macOS. Автоматизирует процесс упаковки Python-кода в готовое к распространению приложение (`.app`) с помощью **PyInstaller**. 
    -   **🆕 ОБНОВЛЕН**: Добавлены все необходимые hidden imports для офлайн режима
    -   Включены модули для работы с сетью (socket, requests, urllib3)
    -   Оптимизирована структура сборки с документацией

-   **`requirements.txt`**: Стандартный файл, перечисляющий все библиотеки Python, необходимые для работы проекта (`Flask`, `pywebview`, `cryptography` и т.д.).

-   **`config.json`**: Файл настроек. Хранит информацию о версии, разработчике, а также URL-адреса внешних сервисов (например, для проверки IP).

-   **`.env`**: Содержит секретный ключ шифрования `SECRET_KEY`. Этот файл критически важен для расшифровки данных.

-   **`generate_key.py`**: Утилита для генерации нового `SECRET_KEY` в файле `.env`. Автоматически создает криптографически стойкий ключ для шифрования данных.

-   **`decrypt_tool.py`**: Инструмент командной строки для просмотра зашифрованных данных без запуска основного приложения.

### Директории данных

-   **`data/`**: Директория для хранения данных.
    -   `servers.json.enc`: Самый важный файл данных. Это **зашифрованный** JSON-файл, содержащий всю информацию о серверах. Шифрование обеспечивает безопасность.
    -   `hints.json`: Простой JSON-файл для хранения команд-шпаргалок.
    -   `merged_*.enc`: Временные файлы для слияния данных при импорте.

-   **`uploads/`**: Пользовательские загрузки.
    -   Иконки серверов в формате PNG
    -   Чеки об оплате и другие документы

-   **📚 `lessons/`**: Образовательные материалы по архитектуре проекта - комплексный курс из 9 уроков, покрывающих все аспекты разработки VPN Server Manager. Включает изучение Flask, шаблонизации, криптографии, многопоточности, GUI с PyWebView и внешних API.

-   **⭐ `memory-bank/`**: Система управления памятью проекта для отслеживания задач, прогресса и рефлексий разработки.

### Интерфейс и статические ресурсы

-   **`static/`**: Статические ресурсы, которые используются в веб-интерфейсе — CSS-стили, картинки, иконки.
    -   **🆕 ОБНОВЛЕН**: Добавлены стили для офлайн режима и анимации загрузки

-   **`templates/`**: HTML-шаблоны, которые Flask использует для генерации веб-страниц. 
    -   **🆕 ОБНОВЛЕН**: `index.html` - добавлен индикатор состояния интернета и улучшенное отображение офлайн режима
    -   **🆕 ОБНОВЛЕН**: `layout.html` - улучшенный JavaScript с обработкой ошибок и анимациями
    -   В `layout.html` добавлен элемент управления масштабированием UI (80%, 90%, 100%)
    -   `settings.html` полностью переработан с новым интерфейсом импорта/экспорта и управления ключами

## Алгоритм Работы Программы

1.  **Запуск**: Пользователь запускает приложение (`VPNServerManager.app`).
2.  **Инициализация**:
    -   `app.py` запускает веб-сервер **Flask** в фоновом потоке. Сервер начинает "слушать" локальный адрес (`http://127.0.0.1:5050`).
    -   Сразу после этого **PyWebView** создает нативное окно приложения.
    -   В этом окне загружается веб-страница по адресу сервера Flask.
    -   Применяется сохранённый масштаб интерфейса из cookies.
3.  **🆕 Проверка интернета**: Приложение проверяет доступность интернета через `socket.create_connection()`.
4.  **Отображение данных**:
    -   Браузер в окне запрашивает главную страницу (`/`).
    -   Flask получает запрос, загружает зашифрованный файл `data/servers.json.enc`, расшифровывает его с помощью ключа из `.env` и передает список серверов в HTML-шаблон `index.html`.
    -   Пользователь видит готовый список своих серверов с индикатором состояния интернета.
5.  **Взаимодействие**: Любое действие пользователя (нажатие кнопки "Редактировать", "Добавить", изменение масштаба, импорт/экспорт) отправляет запрос на сервер Flask, который обрабатывает его, изменяет данные, снова шифрует их и сохраняет обратно в `servers.json.enc`.
6.  **🆕 Обработка ошибок**: При отсутствии интернета или ошибках сети пользователь получает информативные сообщения, а недоступные функции отключаются.
7.  **Экспорт/Импорт**: 
    -   **Экспорт**: Создаются временные файлы или ZIP архивы, которые автоматически удаляются после скачивания
    -   **Импорт**: Внешние файлы проверяются, расшифровываются и объединяются с существующими данными
8.  **Завершение работы**:
    -   Пользователь нажимает на красный крестик закрытия окна.
    -   PyWebView перехватывает это событие и отправляет фоновый запрос на специальный адрес `/shutdown`.
    -   Flask, получив этот запрос, инициирует остановку всего приложения, корректно завершая свою работу и освобождая сетевой порт.

## Безопасность

- Все чувствительные данные (пароли) шифруются алгоритмом **Fernet** (AES-128 + HMAC-SHA256)
- Секретный ключ хранится в файле `.env` и не попадает в Git
- **Поддержка множественных ключей**: Возможность импорта данных с разными ключами шифрования для переноса между установками
- URL-адреса панелей управления маскируются в интерфейсе для защиты от случайных взглядов 
- **Безопасные временные файлы**: Автоматическое удаление временных файлов после экспорта
- **Валидация данных**: Проверка целостности и корректности импортируемых данных
- **🆕 Офлайн безопасность**: Приложение работает безопасно даже без интернета

## Использование Функций Импорта/Экспорта

### Полный Перенос на Другой Компьютер
1. На старом компьютере: **Настройки** → **Полный экспорт** → скачать ZIP архив
2. На новом компьютере: распаковать архив, следовать инструкциям в README.txt

### Импорт Серверов из Другой Установки  
1. Получить `.enc` файл и `SECRET_KEY` с другого компьютера
2. **Настройки** → **Импорт серверов из другой установки**
3. Загрузить `.enc` файл и вставить `SECRET_KEY`
4. Сервера автоматически объединятся с существующими

### Создание Резервной Копии
- **Экспорт Данных**: только серверы
- **Экспорт Ключа**: только ключ шифрования  
- **Полный экспорт**: полная резервная копия с файлами

### Управление ключами шифрования
- **Смена ключа**: Генерация нового ключа с перешифровкой всех данных
- **Проверка соответствия**: Верификация ключа и данных без импорта
- **Автоматическое резервное копирование**: Создание backup перед сменой ключа
