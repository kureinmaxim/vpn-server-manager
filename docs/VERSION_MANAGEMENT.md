## Обзор системы

В проекте VPN Server Manager версии управляются **централизованно** через шаблон конфигурационного файла:

✅ **Единый источник правды** — версия хранится только в шаблоне `config/config.json.template`  
✅ **Автоматическая синхронизация** — скрипты обновляют рабочий конфиг  
✅ **Защита настроек** — рабочий `config.json` не обновляется целиком, только версия  

---

## Схема версионирования

Используется [Semantic Versioning](https://semver.org/) в формате **MAJOR.MINOR.PATCH**:

| Часть | Когда увеличивать | Пример |
|-------|-------------------|--------|
| **MAJOR** | Критические изменения | `4.0.0` → `5.0.0` |
| **MINOR** | Новые функции | `4.0.0` → `4.1.0` |
| **PATCH** | Исправление ошибок | `4.1.0` → `4.1.1` |

## Источник правды

### Где хранится версия

```
config/config.json.template
```

Этот файл:
- ✅ Хранится в Git
- ✅ Содержит актуальную версию для релиза
- ✅ Используется для генерации рабочего конфига

### Ключевые поля

```json
{
  "app_info": {
    "version": "4.0.10",
    "release_date": "26.10.2025",
    "developer": "Куреин М.Н."
  }
}
```

### Локальный файл

`config.json` в корне проекта — это локальный файл настроек. Он должен быть в `.gitignore` (если содержит секреты), или просто не коммититься с изменениями версии вручную. Скрипты синхронизации обновляют в нем версию из шаблона.

## Инструмент bump_version.py

Скрипт для повышения версии в шаблоне.

### Расположение
```
tools/bump_version.py
```

### Примеры использования

#### 🐛 Исправление бага (Patch)
```bash
python tools/bump_version.py --bump patch
# 4.0.10 → 4.0.11
```

#### ✨ Новая функция (Minor)
```bash
python tools/bump_version.py --bump minor
# 4.0.10 → 4.1.0
```

#### Установка конкретной версии
```bash
python tools/bump_version.py --version 4.5.0
```

## Инструмент update_version.py

Скрипт для синхронизации версии между шаблоном и рабочим окружением.

### Расположение
```
tools/update_version.py
```

### Команды

#### Статус версий
Показывает версию в шаблоне и в текущем `config.json`.
```bash
python tools/update_version.py status
```

#### Синхронизация
Копирует версию и метаданные из шаблона в `config.json`, не затрагивая другие настройки.
```bash
python tools/update_version.py sync
```

## Рабочие сценарии

### 🎯 Релиз новой версии

1. **Обновить версию в шаблоне:**
   ```bash
   python tools/bump_version.py --bump patch
   ```

2. **Синхронизировать с рабочим конфигом (для проверки):**
   ```bash
   python tools/update_version.py sync
   ```

3. **Проверить статус:**
   ```bash
   python tools/update_version.py status
   ```

4. **Закоммитить изменения:**
   ```bash
   git add config/config.json.template
   git commit -m "Bump version to 4.0.11"
   ```
