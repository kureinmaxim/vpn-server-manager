cd /Users/olgazaharova/Project/ProjectPython/VPNserverManage && source venv/bin/activate && python3 app.py

## 🚀 Особенности

- **🔐 Безопасность**: Все данные шифруются алгоритмом Fernet (AES-128 + HMAC-SHA256)
- **🔒 PIN-защита**: Дополнительная защита приложения PIN-кодом
- **🖥️ Нативный интерфейс**: PyWebView обеспечивает нативное окно приложения
- **📱 Офлайн режим**: Работает без интернета с индикатором состояния сети
- **🔄 Импорт/Экспорт**: Полная система переноса данных между установками
- **🔑 Управление ключами**: Смена ключей шифрования с автоматической перешифровкой
- **📊 Мониторинг**: Автоматическое отслеживание статуса серверов и платежей
- **🎨 Адаптивный дизайн**: Поддержка светлой и темной темы, масштабирование UI
- **🌐 IP-анализ**: Интеграция с IP2Location для определения владельца IP-адресов
- **📚 Шпаргалка**: Комплексная шпаргалка с командами NGINX, Docker, Systemd и содержанием

## 💻 Установка

### macOS

1. **Скачайте приложение**:
   ```bash
   # Скачайте VPNServer_Installer.dmg
   # Или соберите из исходников:
   git clone https://github.com/your-repo/vpn-server-manager.git
   cd vpn-server-manager
   python3 build_macos.py
   ```

2. **Установите**:
   - Откройте `VPNServer_Installer.dmg`
   - Перетащите `VPNServer.app` в папку Applications
   - При первом запуске: правый клик → "Открыть"

### Из исходников

```bash
# Клонируйте репозиторий
git clone https://github.com/your-repo/vpn-server-manager.git
cd vpn-server-manager

# Создайте виртуальное окружение
python3 -m venv venv
source venv/bin/activate  # На macOS/Linux
# или
venv\Scripts\activate     # На Windows

# Установите зависимости
pip install -r requirements.txt

# Создайте файл .env с ключом шифрования
python3 tools/generate_key.py

# Запустите приложение
python3 app.py
```

## 🔒 PIN-система

Приложение защищено PIN-кодом для дополнительной безопасности:

### Первый запуск
При первом запуске вы увидите экран "Доступ ограничен". У вас есть два варианта:
- **Создать новый PIN**: Нажмите "Первый запуск" → "Создать новый PIN-код"
- **Импортировать архив**: Нажмите "Первый запуск" → "Импортировать архив"

### Вход в приложение
После настройки PIN используйте один из способов:
- **Двойной клик по имени разработчика**: Внизу страницы найдите имя разработчика в футере и дважды кликните по нему
- **Через настройки**: Если вы уже вошли в систему, можете изменить PIN в разделе "Настройки"

### Безопасность
- **Минимальная длина**: PIN должен содержать минимум 4 символа
- **Защита от перебора**: После 2 неудачных попыток вход блокируется на 30 секунд
- **Автоматический счетчик**: При блокировке отображается обратный отсчет времени
- **Хранение**: PIN хранится в зашифрованном виде

### Экспорт и импорт с PIN
- **Экспорт**: При "Полном экспорте" PIN автоматически включается в архив
- **Импорт**: При импорте архива PIN извлекается и устанавливается автоматически
- **Показ PIN**: При импорте PIN отображается в отдельном окне для записи

### Защита информации
- **Маскирование URL**: Панели управления скрыты от случайных взглядов
- **Безопасные пароли**: Автоматическое скрытие/показ паролей
- **Временные файлы**: Автоматическое удаление после экспорта

### Управление ключами
- **Смена ключа**: Безопасная замена с перешифровкой
- **Проверка соответствия**: Верификация ключа и данных
- **Резервное копирование**: Автоматические backup при смене ключей

## 📤📥 Импорт/Экспорт данных

### Экспорт данных

#### 1. Экспорт данных серверов
- Скачивает только `.enc` файл с серверами
- Быстрое создание резервной копии

#### 2. Экспорт ключа шифрования
- Создает `.env` файл с SECRET_KEY
- Для переноса ключа между установками

#### 3. Полный экспорт
- ZIP архив со всеми данными
- Включает: серверы, ключ, PIN, файлы, инструкции
- Готов для полного переноса

### Импорт данных

#### Импорт из другой установки
1. Получите `.enc` файл и `SECRET_KEY`
2. Загрузите файл и вставьте ключ
3. Сервера автоматически объединятся
4. Дубликаты предотвращаются автоматически

#### Полный перенос
1. Скачайте полный экспорт с другого компьютера
2. Распакуйте архив
3. Следуйте инструкциям в README.txt

## 🆘 Поддержка

### Часто задаваемые вопросы

**Q: Приложение не запускается**
A: Проверьте наличие файла `.env` с SECRET_KEY. Используйте `python3 tools/generate_key.py` для создания.

**Q: Не могу импортировать данные**
A: Убедитесь, что ключ шифрования соответствует файлу данных. Используйте "Проверить соответствие" в настройках.

**Q: Потерял ключ шифрования**
A: Без ключа данные невозможно восстановить. Всегда делайте резервные копии через "Полный экспорт".
